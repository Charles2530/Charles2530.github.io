yaml笔记

概念

YAML 是 “YAML Ain’t a Markup Language”（YAML 不是一种标记语言）的递归缩写,是一个类似 XML、JSON 的非标记性语言。YAML 强调以数据为中心，并不是以标识语言为重点，使用空白，缩进，分行组织数据，从而使得表示更加简洁易读。因而 YAML 本身的定义比较简单，号称“一种人性化的数据格式语言”。
YAML的配置文件后缀为.yml或.yaml

语法

  • 大小写敏感
  • 缩进表示层级关系
  • 不允许tab, 只运行空格
  • 缩进长度没有限制，只要元素对齐就表示这些元素属于一个层级
  • # 表示注释
  • 字符串可以不用引号标注

数据类型

  • 对象：键值对的集合, 又名映射、哈希、字典
  • 数组： 一组按序列排的值， 又名序列，列表
  • 存量：单个的、不可分的值

三种数据结构

对象（字典）

键是唯一的, 值可重复。使用冒号（：）表示键值对，同一缩进的所有键值对属于一个map

数组（列表）

一组按序列排的值，使用连字符（-）表示。

纯量

数据最小的单位，不可以再分割。存量可以说是任意如: 字符串、布尔值、整数、浮点数、Null、时间、日期。其实也可以是说写在yml中的就是纯量

适用场景

  • 在脚步语言中使用，实现简单，解析成本低；
  • 序列化；
  • 编程时写配置文件，比xml快，比ini文档功能更强。
  • YAML是专门用于写配置文件的语言，非常简洁和强大，远比JSON格式方便。
