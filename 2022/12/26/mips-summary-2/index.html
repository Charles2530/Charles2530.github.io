0.最重要最重要的内容

如果你不知道进一步学习MIPS语言的正确途径，可能会迷失方向。主要推荐三个途径。首先是课程资料中，高老师的MIPS-C指令集，对于了解每个语句的细节（例如立即数的范围等）很有意义。其次是MARS HELP，在上面你可以看到关于syscall和其他关于MARS的内容。最后是MARS.jar，正所谓实践出真知，编辑器也是学习工具的一部分。

（非常建议先浏览一遍MIPS-C指令集。有条件建议打印一份，方便查阅。）

好的资料还有很多，但对于速通选手可能更多的资料不一定是更好的选择。确保参考资料的准确性，不要盲目相信其他来历不明的资料（比如这篇（））。

1.战术级

首先强调一些关于汇编程序设计的要点。

  1. 先了解语言的基本结构和特征。

    （边学边用当然可以，但是至少要先学会hello world吧）

  2. 保持良好的代码风格习惯。

  3. 再开始敲代码之前，先写出c语言程序（或者其他语言也行）.

    （如果你没有一次写对的自信，那就一定要先用别的方式实现，不然debug都没有参照）

  4. 积极拆分每个独立功能，分开实现。（多用.macro）

  5. 多写注释。

  6. 多写注释。

2.艺术级

优雅地实现方式会大大提高程序的可读性。如果码完就再也不看，那么请随便写。

  1. 自定义编辑器。

    MARS的默认配色不是很易读。建议把编辑器调到舒适为止。

  2. 遵守寄存器的用法。

    如果不加区分地随便使用寄存器，大家都知道会变成什么样的。

  3. 保持固定的缩进格式。

    我用的缩进是8个空格(可能略长，请找一个看着舒服的)，并且在.macro，if，for的部分会使用额外的行首缩进。我在每行指令的两个值之间也选择使用缩进（也可以不搞，因为tab键也不好按）。像这样：

            add     $s1,    $zero,  $t1
        li      $s2,    -20
        beq     $s1,    $s2,    end_if
        nop
begin_if:
                addi    $s1,    1
                ori     $s1,    $s1,    $s2
                j       end_if
                nop
end_if:
        printInt($s1)
        jal     func
        end                     # This code is example

    整齐的话，读起来也轻松一些

  4. 多打label。

    就像上面那个例子一样，就算打不使用的label也可以让代码更明确。

    我在lable和macro用了不同的命名方式，可能有一点让人不适，请谅解。

  5. 可以打标点也可以不打，但是不要时打时不打。

  6. 多用

    .macro

    。

    除非你就是喜欢蜘蛛网一样错综复杂的代码。

  7. 在macro里做好出入栈

    比如：

    .macro printStr(%str)
        push($a0)
        la      $a0,    %str
        li      $v0,    4
        syscall
        pop($a0)
.endmacro

    可以减少后顾之忧。

    在其他函数里也是一样的。所有用到的寄存器都要在使用前压栈，在使用后弹栈（注意弹栈顺序）。在递归进入深层函数时，要注意保留入参的值（可以选择通过传参，但总先复制到t0，每次用复制进行运算）

    别忘了把栈操作也写成macro。

3.技术级

使用和避免特定操作，可以大幅降低错误率。

  1. 遵守对特定寄存器的回避。

    macro的入参和macro中使用的寄存器重复时，会发生难以定位的错误。所以不要使用macro中用过的寄存器作为入参。

    一种做法是将$a0, $t8, $t9用于macro部分，此后的运算中再也不用这几个寄存器。

  2. 使用更稳定（运行稳定，或者让你自己心态稳定）的指令。

    比如在ori $t0,$0,100和li $t0,100中，优先选择更稳定的后者。

    又如在自加（根本记不住那是什么指令）和addi $t0,$t0,1，优先使用更易于快速理解的后者。

    又如减少使用.ascii，使用.asciiz。

  3. 固定重要变量所对应的寄存器。

    比如循环变量，数据范围，常数，答案等值，优先使用固定的寄存器记录，并不再改变。 记得记录每个寄存器固定成了什么变量。例如：

    scanInt($s1)            #n
scanInt($s2)            #m
li      $s3,    0       #ans
li      $s4,    0       #i
#       $s5             #x
#       $s6             #y

    之后使用的时候就能更明确哪个寄存器存的是什么了。

    在函数里可以把不需要的变量压栈，省出寄存器的位置。

  4. 固定for和if语句的格式。

    比如这样：

    func_for1_begin:
        beq     $s4,    $s1,  func_for1_end     # i!=n
        nop
                # do something
                # do something
                # do something
        addi    $s4,    $s4,    1               # i++
        j       func_for1_begin
func_for1_end:

  5. 封装对存储的访问。

    把load和save指令封装成macro的形式。比如.macro arrayLoad(%des,%src)，记得把macro内用到的寄存器压栈并在读取后弹栈。

  6. 封装输入输出。

    参考Mars Help关于syscall部分的解释。至少应当封装int的输入输出，以及string的输出。因为很有用，比如：
